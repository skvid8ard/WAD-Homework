# Changelog

Все заметные изменения в этом проекте будут документироваться в этом файле.
Формат основан на[Keep a Changelog](https://keepachangelog.com/ru/1.0.0/), 
и этот проект придерживается [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.12.0] - 2026-04-26
### Добавлено
- **Skeleton Loaders**: Внедрены мерцающие заглушки для бокового меню и ленты сообщений. Это улучшило "воспринимаемую производительность" (Perceived Performance) — пользователь понимает структуру данных еще до их загрузки.
- **Enhanced Code Experience**: Блоки кода в Markdown теперь имеют стильную шапку с указанием языка и интерактивную кнопку "Copy", интегрированную с системным буфером обмена.
- **Global Toast Notifications**: Подключена библиотека `react-hot-toast`. Реализована система мгновенной обратной связи: подтверждение копирования, ошибки авторизации и уведомления о сбоях сети.
- **Mobile First Navigation**: Реализован адаптивный Layout с выезжающим (Off-canvas) меню и Blur-эффектом для мобильных устройств.
- **Seamless OAuth Flow**: Полная поддержка Google и GitHub Login. Бэкенд расширен эндпоинтом-трамплином `/authorize` для безопасного редиректа.

### Исправлено
- **UI Overflow**: Полностью переписана структура `Chat.tsx` на Flexbox. Устранены баги с перекрытием сообщений полем ввода и исчезновением скроллбара.
- **Consistency**: Унифицированы стили кнопок по всему приложению. Добавлены микро-интеракции `active:scale` для тактильного отклика.
- **Security**: Исправлена утечка имен методов в OAuth-провайдерах через использование `@classmethod`.

## [0.11.0] - 2026-04-26
### Добавлено
- **Mobile First Layout**: Сайдбар теперь адаптивный (Off-canvas). На мобильных устройствах он скрыт и вызывается через "гамбургер-меню" (☰), на десктопах — статичен.
- **Copy to Clipboard**: В блоках кода Markdown появилась функциональная шапка с указанием языка и кнопкой "Copy" для мгновенного копирования в буфер обмена.
- **Toast Notifications**: Интегрирована библиотека `react-hot-toast`. Добавлены уведомления об успешном копировании кода и глобальные оповещения об ошибках сети (404, 500, Connection Refused).
- **Custom Scrollbars**: Реализованы тонкие, адаптивные под тему ползунки прокрутки, которые не портят внешний вид приложения в темном режиме.

### Изменено
- **UI Refactoring**: Отказ от `absolute` позиционирования инпута в пользу `flex-col` структуры. Это решило проблемы с перекрытием сообщений и некорректным скроллом.
- **Centered Content**: Контент чата теперь ограничен шириной `max-w-4xl` и центрирован, что значительно улучшает читаемость на широкоформатных мониторах.
- **Enhanced Micro-interactions**: Кнопки получили физическую отдачу при нажатии (`active:scale`) и плавные переходы (`duration-200`).

## [0.10.1] - 2026-04-26
### Исправлено
- **Axios Interceptor Loop**: Исправлен критический баг бесконечного цикла запросов к `/auth/refresh` при получении 401 ошибки на страницах авторизации (Login). Добавлены исключения для маршрутов логина и рефреша.

### Добавлено
- **PublicRoute**: Внедрен компонент-страж для блокировки доступа к страницам `/login` и `/register` для уже авторизованных пользователей (редирект на `/`).
- **Fetch Wrapper (`fetchWithAuth`)**: Написана кастомная обертка над нативным `fetch` API для поддержки стриминга (SSE), которая эмулирует логику Axios по перехвату 401 ошибки и фоновому обновлению токенов.

## [0.10.0] - 2026-04-26
### Добавлено
- **Stateful Sidebar**: Интеграция `zustand` для глобального управления списком чатов (`chatStore.ts`). Боковое меню динамически обновляется при создании новых бесед.
- **Chat History**: Подгрузка истории сообщений (`GET /chats/{id}/messages`) при переключении между комнатами и автоматическая очистка экрана при выборе "Нового чата".
- **Frictionless UX (Авторизация)**: Улучшен процесс проверки почты. При входе с неподтвержденным email система автоматически отправляет новый код и переводит пользователя на экран ввода без лишних кликов.
- **Reusable Components**: Создан универсальный компонент `<VerificationForm />` для предотвращения дублирования кода (DRY) между `Login.tsx` и `Register.tsx`.
- **Micro-interactions**: Внедрены плавные анимации физического нажатия кнопок (`active:scale-[0.98]`, `active:scale-95`) для улучшения тактильного отклика UI.
- **Secure Logout**: Полноценное завершение сессии. Инвалидация токена на сервере (Redis), очистка локальных чатов в хранилище и перенаправление на экран входа.

### Исправлено
- Исправлен баг, при котором темная/светлая тема на страницах вне `Layout` перекрывалась глобальными стилями Tailwind. Состояние темы вынесено в `themeStore.ts` с персистентностью в `localStorage`.
- Закрыта уязвимость **User Enumeration**: эндпоинты `/auth/resend-code` и `/auth/verify-email` переведены на универсальный поиск `email_or_username` на стороне сервера, чтобы фронтенд не оперировал скрытыми email-адресами пользователей по логину.

##[0.9.0] - 2026-04-26
### Добавлено
- **Стриминг ответов LLM**: Реализовано чтение Server-Sent Events (SSE) на клиенте. Внедрен нативный `fetch` API с `response.body.getReader()` для покадрового рендера текста (Эффект "печатной машинки").
- **Smart Chat UI**: Создан интерфейс чата (Flexbox) с независимой прокруткой ленты сообщений.
- **Auto-resize Textarea**: Поле ввода автоматически увеличивает высоту при наборе длинного текста до максимального лимита.
- **Auto-scroll**: Автоматическая плавная прокрутка ленты (`scrollIntoView`) к последнему сообщению во время потоковой генерации.
- **Безопасный Markdown**: Интеграция `react-markdown` и `remark-gfm` для рендера таблиц, списков и форматирования в ответах нейросети с защитой от XSS.
- **Синтаксическая подсветка**: Внедрен `react-syntax-highlighter` (тема VS Code Dark) для красивого отображения блоков кода, генерируемых LLM.
- **Скрытие скроллбара**: Кастомные CSS-правила `.scrollbar-hide` для эстетичного вида поля ввода.

## [0.8.0] - 2026-04-26

### Добавлено
- **Global Auth State**: Интеграция Zustand с middleware `persist` для синхронизации токена между вкладками и сохранения сессии после F5.
- **Silent Refresh**: Реализован Response Interceptor в Axios, который автоматически обновляет `access_token` через `/auth/refresh` при получении ошибки 401.
- **Protected Routes**: Создан компонент-страж для блокировки доступа к чатам неавторизованным пользователям.
- **Auth UI**: Сверстаны страницы `/login` и `/register` с использованием `framer-motion` (плавное появление карточек).
- **UX Security**: Добавлен функционал переключения видимости пароля («глазик») и клиентская проверка совпадения паролей.
- **Verification Flow**: Реализована двухэтапная регистрация с вводом OTP-кода и таймером повторной отправки (Resend Code) на 60 секунд.

### Исправлено
- **CORS Policy**: Настройки бэкенда обновлены для поддержки `allow_credentials=True` с указанием конкретных доверенных `origins`.
- **Layout Shift**: Анимации вынесены из глобальных контейнеров во внутренние карточки, чтобы предотвратить скачки скроллбара при загрузке.

## [0.7.1] - 2026-04-25

### Изменено
- **Monorepo Architecture**: Проект реструктурирован в монорепозиторий со строгим разделением на независимые директории: `backend/`, `frontend/`, и `ai-models/`.
- Обновлен класс `Settings` (Pydantic): добавлена поддержка динамического поиска файла конфигурации через `SettingsConfigDict(env_file=["../.env", ".env"])` для корректной работы из подпапок.
- Виртуальное окружение Python (`.venv`) и кэш тестов (`.pytest_cache`) изолированы внутри директории `backend/`.

### Исправлено
- Исправлена критическая ошибка сборки C++ ядра `llama-cpp-python` в среде VirtualBox (Out of Memory и несовместимость инструкций процессора). Сборка перенастроена с использованием строгих флагов `GGML` (`-DGGML_NATIVE=OFF -DGGML_AVX=OFF` и др.) и ограничением потоков компилятора (`CMAKE_BUILD_PARALLEL_LEVEL=1`).
- Исправлена ошибка путей при загрузке LLM: бэкенд теперь корректно ищет модели на уровень выше (`../ai-models/`).

### Безопасность
- Обновлен `.gitignore`: добавлено строгое правило исключения весов нейросетей (`ai-models/*.gguf`), чтобы предотвратить блокировку репозитория при случайном коммите файлов размером более 100 МБ.

- Паттерн Singleton для загрузки весов модели в оперативную память при старте FastAPI (через `lifespan`).

## [0.5.0] - 2026-04-25

### Добавлено
- Интеграция OAuth2 (Authorization Code Flow) для провайдеров Google и GitHub.
- Универсальный динамический эндпоинт `POST /auth/oauth/{provider}/login`.
- Новая таблица `oauth_accounts` (1-ко-многим с `users`) для хранения привязанных соцсетей.
- Механизм **Implicit Linking**: автоматическое связывание социального аккаунта с уже существующим пользователем по совпадению email.
- Автоматическая бесшовная регистрация новых пользователей через OAuth2 (is_verified = True, безопасная автогенерация уникального username на базе email и UUID).
- Изолированные автотесты для OAuth2 с использованием `unittest.mock.patch` для подмены внешних HTTP-вызовов к провайдерам.

### Изменено
- Обновлен `auth_service.py` (Оркестратор): переиспользование существующей логики генерации JWT и Redis-сессий для входа через соцсети.
- `httpx.AsyncClient` в провайдерах теперь использует явный `timeout=15.0` для защиты от зависаний при проблемах с сетью.
- Модели SQLAlchemy получили `default=uuid.uuid4` (на уровне Python) в дополнение к `server_default` (на уровне БД) для обеспечения кроссплатформенности тестов (PostgreSQL + SQLite).

### Исправлено
- Добавлено декодирование URL (`urllib.parse.unquote`), чтобы предотвратить ошибку 400 Bad Request от Google при двойном кодировании символов (`%2F`) фронтендом.
- Устранена ошибка `OperationalError: unknown function: gen_random_uuid()` при тестировании связей в in-memory SQLite базе.

## [0.4.1] - 2026-04-24
### Изменено
- **Рефакторинг (Clean Architecture):** `auth_service.py` разделен на независимые слои (SRP): `user_service.py` (SQLAlchemy) и `session_service.py` (Redis). `auth_service.py` теперь выступает в роли Фасада (Оркестратора).
### Исправлено
- Исправлена проблема `RuntimeError: Event loop is closed` в автотестах путем добавления глобального `pytest.ini` с `asyncio_default_fixture_loop_scope = session`.

## [0.4.0] - 2026-04-22
### Добавлено
- Новые колонки в таблице `users`: `email` (уникальная) и `is_verified` (boolean, default=False).
- Pydantic-валидация `EmailStr` для проверки формата почты при регистрации.
- Эндпоинты `/auth/verify-email` и `/auth/resend-code`.
- Интеграция с Redis: использование команды `SETEX` для хранения 6-значных OTP (One-Time Password) кодов со сроком жизни 10 минут.
- Сервис-заглушка `email_service.py` (Mock Sender) для имитации отправки email-сообщений в консоль при локальной разработке.

### Изменено
- Эндпоинт `/auth/login` переведен с JSON на `OAuth2PasswordRequestForm` для поддержки стандартов OAuth2 и нативной интеграции со Swagger UI.
- Реализован "Универсальный логин": авторизация работает как по `username`, так и по `email` в одном поле формы.
- Бизнес-логика логина теперь блокирует доступ (возвращает `403 Forbidden`), если почта пользователя не подтверждена (`is_verified == False`).

### Исправлено
- Исправлена ошибка `NameError` в зависимости `get_current_user`, связанная с некорректным переходом на UUID.

## [0.3.1] - 2026-04-21
### Добавлено
- Инфраструктура для автотестов (`pytest`, `pytest-asyncio`, `httpx`).
- In-memory база данных SQLite (`aiosqlite`) для изолированного и быстрого тестирования.

## [0.3.0] - 2026-04-21
### Добавлено
- Pydantic-схемы для валидации входных данных пользователя (`UserCreate`, `UserResponse`).
- Сложная валидация пароля (регулярные выражения для проверки регистра, цифр и спецсимволов).
- Эндпоинты `/auth/register`, `/auth/login`, `/auth/refresh`, `/auth/logout`.
- JWT токены (Access и Refresh).
- Зависимость `get_current_user` для защиты эндпоинтов.
- Сохранение `jti` (состояния токена) в Redis со сроком жизни 30 дней.
- Инвалидация сессий в Redis через `scan_iter` для логаута.
- Настройка CORS в `main.py`.

### Изменено
- Тип первичного ключа (`id`) во всех моделях изменен с `Integer` на `UUIDv4` (`gen_random_uuid()` на уровне PostgreSQL).
- Механизм хеширования паролей переведен на `Argon2id` (вместо `bcrypt`).

### Security (Безопасность)
- Refresh токены теперь доставляются исключительно через `Set-Cookie` с флагами `HttpOnly` и `SameSite`.
- Централизованная конфигурация `secure` флага для кук (автоматическое переключение `True/False` в зависимости от `ENVIRONMENT`).
- Внедрен механизм JWT Rotation для защиты от перехвата сессий (replay attacks).
