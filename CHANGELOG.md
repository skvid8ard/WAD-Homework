# Changelog

Все заметные изменения в этом проекте будут документироваться в этом файле.
Формат основан на[Keep a Changelog](https://keepachangelog.com/ru/1.0.0/), 
и этот проект придерживается [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.7.1] - 2026-04-25

### Изменено
- **Monorepo Architecture**: Проект реструктурирован в монорепозиторий со строгим разделением на независимые директории: `backend/`, `frontend/`, и `ai-models/`.
- Обновлен класс `Settings` (Pydantic): добавлена поддержка динамического поиска файла конфигурации через `SettingsConfigDict(env_file=["../.env", ".env"])` для корректной работы из подпапок.
- Виртуальное окружение Python (`.venv`) и кэш тестов (`.pytest_cache`) изолированы внутри директории `backend/`.

### Исправлено
- Исправлена критическая ошибка сборки C++ ядра `llama-cpp-python` в среде VirtualBox (Out of Memory и несовместимость инструкций процессора). Сборка перенастроена с использованием строгих флагов `GGML` (`-DGGML_NATIVE=OFF -DGGML_AVX=OFF` и др.) и ограничением потоков компилятора (`CMAKE_BUILD_PARALLEL_LEVEL=1`).
- Исправлена ошибка путей при загрузке LLM: бэкенд теперь корректно ищет модели на уровень выше (`../ai-models/`).

### Безопасность
- Обновлен `.gitignore`: добавлено строгое правило исключения весов нейросетей (`ai-models/*.gguf`), чтобы предотвратить блокировку репозитория при случайном коммите файлов размером более 100 МБ.

## [0.7.0] - 2026-04-25

### Добавлено
- Инициализирован Frontend проекта на современном стеке: React 18 + Vite + TypeScript.
- Заложена масштабируемая архитектура клиентской части (адаптированный Feature-Sliced Design: `pages`, `widgets`, `shared`).
- Подключен и настроен новейший Tailwind CSS v4 (подход CSS-first, без `postcss.config.js`).
- Настроена клиентская маршрутизация с помощью `react-router-dom` v6 (использование `Layout` и `<Outlet />`).
- Сверстан глобальный `Layout` приложения: левая боковая панель (Sidebar) для списка чатов и основная рабочая область.
- Добавлена библиотека иконок `lucide-react`.
- Реализована встроенная поддержка темной и светлой темы. Темная тема включена по умолчанию на уровне `index.html` для предотвращения белых вспышек (FOUC) при загрузке SPA.

## [0.6.0] - 2026-04-25

### Добавлено
- Интеграция локальной нейросети через `llama-cpp-python`.
- Паттерн Singleton для загрузки весов модели в оперативную память при старте FastAPI (через `lifespan`).
- Изоляция синхронного C++ кода генерации токенов в отдельных потоках через `asyncio.to_thread` (предотвращение блокировки Event Loop).
- Потоковая передача ответов нейросети (Server-Sent Events / StreamingResponse).
- Stateful-чаты: создание комнат, сохранение истории переписки в PostgreSQL.
- Динамическое формирование контекстного окна (памяти) для LLM из базы данных.

### Безопасность
- Защита от IDOR (Insecure Direct Object Reference) на всех эндпоинтах `/chats/{id}/*` через проверку `user_id`.

## [0.5.0] - 2026-04-25

### Добавлено
- Интеграция OAuth2 (Authorization Code Flow) для провайдеров Google и GitHub.
- Универсальный динамический эндпоинт `POST /auth/oauth/{provider}/login`.
- Новая таблица `oauth_accounts` (1-ко-многим с `users`) для хранения привязанных соцсетей.
- Механизм **Implicit Linking**: автоматическое связывание социального аккаунта с уже существующим пользователем по совпадению email.
- Автоматическая бесшовная регистрация новых пользователей через OAuth2 (is_verified = True, безопасная автогенерация уникального username на базе email и UUID).
- Изолированные автотесты для OAuth2 с использованием `unittest.mock.patch` для подмены внешних HTTP-вызовов к провайдерам.

### Изменено
- Обновлен `auth_service.py` (Оркестратор): переиспользование существующей логики генерации JWT и Redis-сессий для входа через соцсети.
- `httpx.AsyncClient` в провайдерах теперь использует явный `timeout=15.0` для защиты от зависаний при проблемах с сетью.
- Модели SQLAlchemy получили `default=uuid.uuid4` (на уровне Python) в дополнение к `server_default` (на уровне БД) для обеспечения кроссплатформенности тестов (PostgreSQL + SQLite).

### Исправлено
- Добавлено декодирование URL (`urllib.parse.unquote`), чтобы предотвратить ошибку 400 Bad Request от Google при двойном кодировании символов (`%2F`) фронтендом.
- Устранена ошибка `OperationalError: unknown function: gen_random_uuid()` при тестировании связей в in-memory SQLite базе.

## [0.4.1] - 2026-04-24
### Изменено
- **Рефакторинг (Clean Architecture):** `auth_service.py` разделен на независимые слои (SRP): `user_service.py` (SQLAlchemy) и `session_service.py` (Redis). `auth_service.py` теперь выступает в роли Фасада (Оркестратора).
### Исправлено
- Исправлена проблема `RuntimeError: Event loop is closed` в автотестах путем добавления глобального `pytest.ini` с `asyncio_default_fixture_loop_scope = session`.

## [0.4.0] - 2026-04-22
### Добавлено
- Новые колонки в таблице `users`: `email` (уникальная) и `is_verified` (boolean, default=False).
- Pydantic-валидация `EmailStr` для проверки формата почты при регистрации.
- Эндпоинты `/auth/verify-email` и `/auth/resend-code`.
- Интеграция с Redis: использование команды `SETEX` для хранения 6-значных OTP (One-Time Password) кодов со сроком жизни 10 минут.
- Сервис-заглушка `email_service.py` (Mock Sender) для имитации отправки email-сообщений в консоль при локальной разработке.

### Изменено
- Эндпоинт `/auth/login` переведен с JSON на `OAuth2PasswordRequestForm` для поддержки стандартов OAuth2 и нативной интеграции со Swagger UI.
- Реализован "Универсальный логин": авторизация работает как по `username`, так и по `email` в одном поле формы.
- Бизнес-логика логина теперь блокирует доступ (возвращает `403 Forbidden`), если почта пользователя не подтверждена (`is_verified == False`).

### Исправлено
- Исправлена ошибка `NameError` в зависимости `get_current_user`, связанная с некорректным переходом на UUID.

## [0.3.1] - 2026-04-21
### Добавлено
- Инфраструктура для автотестов (`pytest`, `pytest-asyncio`, `httpx`).
- In-memory база данных SQLite (`aiosqlite`) для изолированного и быстрого тестирования.
- Переопределение зависимостей FastAPI (`dependency_overrides`) для мокирования сессий БД.
- Покрытие тестами модуля авторизации (`test_auth.py`):
  - Успешная регистрация и валидация схемы паролей (через `pytest.mark.parametrize`).
  - Комплексный флоу логина и обновления сессии (проверка генерации токенов и установки безопасных HTTP-Only Cookie).

## [0.3.0] - 2026-04-21
### Добавлено
- Pydantic-схемы для валидации входных данных пользователя (`UserCreate`, `UserResponse`).
- Сложная валидация пароля (регулярные выражения для проверки регистра, цифр и спецсимволов).
- Эндпоинты `/auth/register`, `/auth/login`, `/auth/refresh`, `/auth/logout`.
- JWT токены (Access и Refresh).
- Зависимость `get_current_user` для защиты эндпоинтов.
- Сохранение `jti` (состояния токена) в Redis со сроком жизни 30 дней.
- Инвалидация сессий в Redis через `scan_iter` для логаута.
- Настройка CORS в `main.py`.

### Изменено
- Тип первичного ключа (`id`) во всех моделях изменен с `Integer` на `UUIDv4` (`gen_random_uuid()` на уровне PostgreSQL).
- Механизм хеширования паролей переведен на `Argon2id` (вместо `bcrypt`).

### Security (Безопасность)
- Refresh токены теперь доставляются исключительно через `Set-Cookie` с флагами `HttpOnly` и `SameSite`.
- Централизованная конфигурация `secure` флага для кук (автоматическое переключение `True/False` в зависимости от `ENVIRONMENT`).
- Внедрен механизм JWT Rotation для защиты от перехвата сессий (replay attacks).
