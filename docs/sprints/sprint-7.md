# Sprint 7: Frontend Фундамент и UI-Кит (Layout & Theme)

**Статус:** Завершен  
**Версия:** v0.7.0  

## Цели спринта:
Инициализировать Single Page Application (SPA) для клиентской части приложения. Настроить современные инструменты сборки, строгую типизацию, систему маршрутизации и создать базовый "скелет" пользовательского интерфейса (Layout) с поддержкой темной/светлой темы.

## Принятые архитектурные решения (Best Practices):

1. **Сборщик и Фреймворк:**
   - Выбран стек **Vite + React + TypeScript**. Vite обеспечивает мгновенный старт сервера разработки (HMR) благодаря использованию нативных ES-модулей, что решает проблему долгих сборок классического Webpack. TypeScript гарантирует строгую типизацию контрактов между бэкендом и фронтендом.

2. **Адаптированный Feature-Sliced Design (FSD):**
   - Фронтенд разделен на логические слои: `app` (инициализация), `pages` (экраны), `widgets` (крупные блоки, такие как Layout/Sidebar), `shared` (UI-кит и утилиты). Это предотвращает превращение проекта в монолит.

3. **Стилизация (Tailwind CSS v4):**
   - Использована новейшая мажорная версия Tailwind CSS (v4) с подходом "CSS-first". Конфигурация пишется напрямую в `index.css` (`@import "tailwindcss";`), отказавшись от `postcss` конфигураций, что делает рендеринг стилей в 5 раз быстрее.

4. **Предотвращение FOUC (Flash of Unstyled Content):**
   - Темная тема реализована через глобальный класс `dark` на теге `<html>` прямо в `index.html`. Это гарантирует, что браузер отрисует темный фон еще до загрузки и выполнения JavaScript-бандла, избегая "белых вспышек" при загрузке страницы.

5. **SPA Роутинг:**
   - Настроен `react-router-dom` (v6). Использован паттерн `Outlet` внутри компонента `Layout`. Это позволяет перерисовывать только центральную часть приложения (чаты, настройки) без перезагрузки бокового меню (Sidebar).

## Результат:
Развернут фронтенд-сервер. Сверстан строгий, современный Layout с сайдбаром, иконками (`lucide-react`) и рабочим переключателем светлой/темной темы. Архитектура полностью готова к интеграции бизнес-логики и связи с API бэкенда.
