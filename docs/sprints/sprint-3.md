# Sprint 3: Аутентификация, Безопасность и Сессии

**Статус:** Завершен
**Версия:** v0.3.0

## Цели спринта:
Реализовать безопасную систему регистрации, аутентификации, авторизации и управления сессиями (JWT + Redis).

## Принятые архитектурные решения (Best Practices):
1. **Предотвращение IDOR (Insecure Direct Object Reference):** 
   * Отказ от последовательных `int` ID в пользу `UUIDv4`.
   * Генерация UUID делегирована ядру PostgreSQL (`server_default=text("gen_random_uuid()")`) для избежания race conditions в SQLAlchemy.
2. **Криптография:** 
   * Использован алгоритм `Argon2id` для хеширования паролей как текущий золотой стандарт (победитель PHC), более устойчивый к GPU-атакам, чем bcrypt.
3. **Безопасность токенов (Stateless + Stateful):**
   * Внедрен механизм **JWT Rotation**. 
   * Access Token (короткоживущий, 30 минут) передается в JSON-ответе.
   * Refresh Token (долгоживущий, 30 дней) содержит уникальный `jti` и передается строго через `HttpOnly`, `SameSite=lax` cookie с ограничением пути (`path=/auth/refresh`) для защиты от XSS и CSRF.
   * Состояние сессий (связка `user_id` + `jti`) хранится в Redis.
4. **Валидация данных:**
   * Строгая валидация сложности пароля (заглавные, строчные, цифры, спецсимволы) через `pydantic` схемы на уровне бэкенда.
5. **Инвалидация сессий (Logout):**
   * Реализовано уничтожение всех сессий пользователя. Для обхода ограничения однопоточности Redis использован неблокирующий итератор `scan` вместо `keys`.
6. **Автоматизированное тестирование (TDD/BDD подход):**
   * Развернута среда тестирования на базе `pytest` и `httpx` (AsyncClient).
   * Реализован паттерн "Dependency Override": боевая PostgreSQL подменяется на `in-memory` SQLite для высокой скорости выполнения тестов без влияния на персистентные данные.
   * Использована фикстура уровня сессии для автоматического создания/удаления таблиц (`Base.metadata.create_all`).
   * Внедрена параметризация (`@pytest.mark.parametrize`) для эффективного тестирования edge-кейсов валидации паролей.
