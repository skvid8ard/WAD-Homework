# Sprint 8: Авторизация, Глобальный стейт и UX

**Статус:** Завершен ✅
**Версия:** v0.8.0

## Цели спринта:
Связать фронтенд с системой авторизации бэкенда, обеспечив безопасность и плавность интерфейса.

## Принятые архитектурные решения:

1. **Zustand vs Context API**:
   Выбран Zustand из-за возможности доступа к состоянию вне React-компонентов (в интерцепторах Axios), что критично для логики обновления токенов.

2. **OAuth2 Form Data**:
   Для эндпоинта `/login` использован `URLSearchParams`, так как FastAPI `OAuth2PasswordRequestForm` ожидает формат `application/x-www-form-urlencoded`, а не JSON.

3. **Atomic Interceptors**:
   Логика рефреша инкапсулирована в `api.ts`. Это позволяет остальной части приложения (чатам, профилю) даже не знать о существовании токенов — они просто делают запросы, а клиент сам решает вопросы авторизации "под капотом".

4. **Двухэтапный UX регистрации**:
   Вместо перенаправления на новую страницу верификации, использован условный рендеринг внутри `Register.tsx`. Это сохраняет контекст (email пользователя) и ускоряет процесс.

5. **Framer Motion Scaling**:
   Использована комбинация `opacity` и `scale (0.85 -> 1.0)`, что создает эффект "выныривания" карточки, делая интерфейс более отзывчивым.

## Результаты тестирования:
- [x] При получении 401 ошибки запрос на рефреш уходит автоматически.
- [x] Повторный (изначальный) запрос выполняется успешно после получения нового токена.
- [x] Таймер Resend Code корректно блокирует кнопку на 60 секунд.
- [x] Неавторизованный пользователь при попытке зайти на `/` перенаправляется на `/login`.
