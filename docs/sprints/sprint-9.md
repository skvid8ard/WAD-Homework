Версия: v0.9.0 | Статус: Завершен ✅
🎯 Цели спринта:
Разработать интерактивный интерфейс чата, поддерживающий отображение Markdown, и реализовать механизм чтения потоковых ответов от LLM в реальном времени.
🏗 Ключевые архитектурные решения:
Fetch API vs Axios для SSE:
Решение: Использование нативного fetch с интерфейсом ReadableStream (getReader()).
Почему?: Axios по умолчанию буферизирует ответ до полного завершения запроса. Для Server-Sent Events (SSE) нам нужно обрабатывать каждый "чанк" (кусочек текста) мгновенно. fetch позволяет "стримить" данные прямо в UI, создавая эффект печатающейся нейросети.
Безопасный рендер Markdown:
Решение: Связка react-markdown + remark-gfm + react-syntax-highlighter.
Почему?: Нейросеть отдает сырой текст. Прямая вставка через innerHTML опасна (XSS). Библиотеки парсят текст в безопасные React-компоненты, а хайлайтер подменяет стандартные теги <code> на стилизованные блоки с подсветкой синтаксиса.
Управление состоянием прокрутки:
Решение: Паттерн "Invisible Anchor" (Невидимый якорь) с использованием useRef и scrollIntoView.
Почему?: При добавлении новых букв в поток высота контейнера растет. Чтобы пользователь видел свежий текст, мы программно "примагничиваем" экран к нижнему якорю при каждом обновлении стейта сообщений.
