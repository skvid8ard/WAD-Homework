# Sprint 4: Верификация Email и Универсальный логин

**Статус:** Завершен
**Версия:** v0.4.0

## Цели спринта:
Обеспечить проверку подлинности email-адресов пользователей и улучшить UX авторизации.

## Принятые архитектурные решения (Best Practices):
1. **Хранение OTP в In-Memory БД:** 
   * 6-значные коды верификации хранятся не в PostgreSQL, а в Redis (`SETEX`). Это избавляет основную БД от мусорных данных и автоматически решает проблему истечения срока действия кода (TTL 600 секунд) силами самой NoSQL базы.
2. **Универсальный логин (OAuth2):** 
   * Использование `OAuth2PasswordRequestForm` вместо кастомных JSON-схем. Запрос `or_(User.username == form_data.username, User.email == form_data.username)` позволяет пользователю гибко выбирать способ авторизации, при этом сохраняя совместимость со Swagger UI и стандартами OpenAPI.
3. **Защита от Unverified аккаунтов (Edge Cases):**
   * Внедрена блокировка входа для неподтвержденных пользователей.
   * Реализован эндпоинт `/resend-code` для повторной отправки кода. Это решает проблему пользователей, чьи коды сгорели по истечении времени, предотвращая появление "мертвых" email-адресов в базе, которые навсегда заблокированы для повторной регистрации.
4. **Mocking внешних сервисов:**
   * Добавлен `email_service` с логированием в консоль. Это позволяет разрабатывать и тестировать логику без затрат на реальные SMTP-серверы и без риска попадания в спам-листы на этапе разработки.
