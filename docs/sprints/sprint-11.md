# Sprint 11: Полишинг и Адаптивность
Версия: v0.11.0 | Статус: Завершен ✅
🎯 Цели спринта:
Адаптировать приложение под мобильные устройства, внедрить систему уведомлений и довести визуальную составляющую до уровня премиального продукта.
🏗 Ключевые архитектурные решения:
Адаптивный Off-canvas Sidebar:
Решение: Комбинация fixed позиционирования для мобильных и relative для десктопа с использованием Tailwind-префиксов md:.
Почему?: Позволяет использовать один и тот же DOM-узел для меню, экономя память. На мобильных устройствах меню выезжает поверх чата с эффектом Backdrop Blur, на десктопах — занимает статичное место слева.
Интерактивные блоки кода:
Решение: Кастомная обертка над Markdown-компонентом code.
Почему?: Добавление кнопки "Copy" и индикатора языка программирования значительно улучшает опыт разработчика (Developer Experience), превращая чат в полноценного помощника по кодингу.
Глобальный Toast-провайдер:
Решение: Интеграция react-hot-toast.
Почему?: Дает неблокирующую обратную связь (например, подтверждение копирования или сообщение об ошибке сети). Провайдер вынесен в корень App.tsx, что позволяет вызывать уведомления даже из Axios-интерцепторов вне React-компонентов.
Сенсорная оптимизация:
Решение: Анимации active:scale и transition-all.
Почему?: Создает эффект "физического нажатия" кнопок. Это критично для мобильных интерфейсов, где пользователю важно получить визуальный отклик на касание пальцем.
