Помогай мне пошагово, пиши чистый код и объясняй ключевые моменты. Используй асинхронность (async/await) везде, где это возможно.
Я разрабатываю веб-приложение чат для локальной нейронной модели. Используется qwen.gguf и llama-cpp-python (from llama_cpp import Llama)
Допустимые упрощения
Разрешается не отправлять полную историю переписки модели при каждом новом запросе.
Аутентификация и безопасность
Регистрация с использованием логина и пароля; логин должен быть уникальным.
Пароли должны храниться в базе данных в хешированном виде (например, bcrypt, argon2).
Вход через GitHub OAuth должен поддерживаться наряду с логином по паролю (или как альтернатива ему).
Наличие HTTP API, защищенного с помощью JWT (access-токены).
Сессии обновления (refresh-токены или эквивалентные записи сессий) должны храниться в Redis со сроком жизни (TTL) 30 дней.
API
REST (или аналогичный) API, построенный на FastAPI.
Эндпоинты для авторизации, чатов, сообщений и взаимодействия с LLM в соответствии с вашим проектом.
Валидация JWT на защищенных маршрутах. »

### Шаг 3: Слой Безопасности (Auth Service)
**Цель:** JWT, хеширование паролей и Refresh-токены в Redis.

*   **Промпт для ИИ:**
    > «Реализуй `auth_service.py`. Мне нужна регистрация и логин. 
    > 1. Хеширование паролей через `passlib` (bcrypt). 
    > 2. Генерация Access JWT и Refresh JWT. 
    > 3. Сохранение Refresh токенов в Redis на 30 дней.
    > 4. Функция-зависимость (Dependency) для FastAPI, которая извлекает текущего пользователя из JWT и проверяет его.»

### Шаг 6: API Контроллеры (Routes)
**Цель:** Создать эндпоинты для чатов.

*   **Промпт для ИИ:**

    > 2. `GET /chats` — список чатов юзера.
    > 3. `POST /chats/{id}/messages` — отправить сообщение (сохранить в БД, вызвать LLM Service, сохранить ответ LLM в БД, вернуть ответ).
